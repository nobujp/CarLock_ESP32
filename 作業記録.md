# CarLock ESP32 作業記録

## ハードウェア構成（確認済み）

- **ボード**: Aideepen ESP32 4chリレーモジュール
- **通信**: BLE (Bluetooth Low Energy) Nordic UART Service
- **操作アプリ**: Serial Bluetooth Terminal (Android) / BlueSee (iOS)
- **シリアル速度**: 115200 bps / COM8

### リレーピン配置（スキャンテストで確認済み）

| リレー | GPIO | 役割 |
|--------|------|------|
| RELAY_1 | 23 | ロック |
| RELAY_2 | 5 | アンロック |
| RELAY_3 | 4 | ハザード / アンサーバック |
| RELAY_4 | 13 | 予備 |

- **動作極性**: active LOW（LOW=ON, HIGH=OFF）

---

## コマンド一覧

| 送信コマンド | 動作 |
|------------|------|
| `1234-L` | ロック（RELAY_1 500ms + ハザード1回） |
| `1234-U` | アンロック（RELAY_2 500ms + ハザード2回） |
| `1234-H` | ハザード5秒間点滅 |
| `1234-R4` | RELAY_4 500ms（予備） |
| `1234-STATUS` | 状態確認（IDLE/BUSY） |

## トラブルと解決策

### 1. 起動時に全リレーがON
- **原因**: `RELAY_ON=HIGH, RELAY_OFF=LOW` の極性設定が逆だった
- **解決**: `RELAY_ON=LOW, RELAY_OFF=HIGH` に変更（active LOW確認）

### 2. 起動時にRELAY_3だけONのまま
- **原因**: `pinMode(OUTPUT)` にする瞬間ピンがLOWになりリレーがONになった
- **解決**: `digitalWrite(HIGH)` → `pinMode(OUTPUT)` の順番に変更

### 3. BLEコマンドを受信しているのにリレーが動かない
- **原因**: BLEは別タスク（Core0）で動作するため、コールバック内の `digitalWrite()` が物理的に反映されなかった
- **解決**: コールバック内では `pendingCmd` フラグを立てるだけにして、`loop()` 内でGPIO操作を実行

### 4. ロック/アンサーバックのタイミングがずれる
- **原因**: ロック完了後にハザードを開始していたため遅延が発生
- **解決**: ロック/アンロックリレーと同時にハザードも開始
